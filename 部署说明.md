# 部署说明

这个项目现在支持两种用法：

1. 没填写 Supabase 配置时：还是存在自己电脑浏览器里，适合本机测试。
2. 填写 Supabase 配置后：案例、图片、视频会保存到 Supabase 云端，别人打开线上网页也能看到同一份案例库。

## 一、先建 Supabase 云端数据

1. 打开 Supabase。
2. 进入你的项目。
3. 打开左侧 `SQL Editor`。
4. 打开本项目里的 `supabase-setup.sql`。
5. 如果要改管理员账号或密码，修改 SQL 最下面的 `app_admin_logins` 那一行。
6. 复制整份 SQL，粘贴到 Supabase 的 SQL Editor 里运行。

这个 SQL 会自动创建：

- 案例表
- 管理员账号表
- 图片/视频存储桶
- 权限规则：别人只能看，管理员登录后才能新增、编辑、删除

## 二、填写 Supabase 配置

1. 在 Supabase 项目里打开 `Project Settings`。
2. 找到 `API`。
3. 复制 `Project URL`。
4. 复制 `anon public` key。
5. 打开本项目里的 `config.js`。
6. 填成这样：

```js
window.SUPABASE_CONFIG = {
  url: "你的 Project URL",
  anonKey: "你的 anon public key",
};
```

注意：`anon public key` 是前端公开用的，不是服务端密钥。不要把 `service_role` key 填进去。

## 三、打开管理员登录

网页发布后，点右上角 `管理员登录`，输入管理员账号和密码。

当前管理员账号是 `CCCC`。密码不会明文写在代码里，Supabase 里保存的是密码校验值。

## 四、发布到 GitHub Pages

1. 在 GitHub 新建一个仓库。
2. 把这个文件夹里的文件上传到仓库。
3. 打开仓库的 `Settings`。
4. 打开 `Pages`。
5. Source 选择 `Deploy from a branch`。
6. Branch 选择 `main`，文件夹选择 `/root`。
7. 保存后等一会，GitHub 会给你一个网址。

以后你把网址发给别人，别人就能同时看到 Supabase 云端里的案例。
